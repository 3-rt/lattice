## Design Context

### Users
Lattice serves mixed-skill teams working with Claude Code, OpenClaw, and Codex. The dashboard needs to support both technical operators who care about orchestration fidelity and non-technical collaborators who need to dispatch work, understand agent state, and run workflows without deciphering protocol details.

The primary jobs are:
- dispatch tasks to the right agent
- understand what the system is doing right now
- build and run workflows that coordinate multiple agents

### Brand Personality
The product should feel capable, direct, and welcoming. It should read like a serious control plane, not an intimidating developer toy.

Three-word personality:
- assured
- technical
- approachable

Emotional goals:
- confidence during live activity
- clarity during setup and triage
- approachability for first-time and non-technical users

### Aesthetic Direction
Primary theme is dark. The overall direction is `Orbital Console`: a dark, atmospheric operations hub with disciplined contrast, signal-driven accent colors, and stronger visual energy in live orchestration views.

Surface-specific tone:
- Live Flow should feel like mission control: ambient, active, and legible at a glance.
- Workflow Builder should feel more technical and exact: sharper, more structured, less atmospheric.
- Agent, task, and navigation surfaces should stay friendlier and plainer so mixed-skill users are not pushed into a hyper-technical UI.

Avoid:
- generic AI neon-on-black styling
- purple-heavy gradients
- decorative glow used everywhere
- visual noise that makes the interface feel exclusive to engineers

### Design Principles
1. Make system state obvious without requiring technical background.
2. Concentrate visual drama in live orchestration surfaces and keep operational pages calmer.
3. Use color as signal, not decoration.
4. Preserve precision in builder/editor surfaces through tighter structure and cleaner controls.
5. Keep the product welcoming by pairing strong hierarchy with plainspoken labels and readable layouts.
